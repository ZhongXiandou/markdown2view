# A4 文档分页拆分引擎设计（splitBlockByHeight）

> 本文档定义 A4 文档模式下「块内容按行/按单元跨页拆分」的通用抽象方案，目标是消除「整段换页留大空白」，并把现有表格分页逻辑收敛进统一引擎。
>
> 状态：**设计评审稿（未实现）**。评审通过后再分阶段落地。
>
> 涉及文件：`src/modes/document/documentModel.ts`、`src/lib/useBlockHeights.ts`、`src/modes/document/DocumentMode.tsx`、`src/engine/utils/markdownParser.ts`（仅读取其渲染结构，不改引擎）。

---

## 1. 背景与目标

### 1.1 要解决的问题

1. **底部冗余空间 / 莫名换页**：当前除表格外所有块都是**原子**的，任一块放不下就整块推到下一页，留下大片空白。
2. **长文本无法按行跨页**：长段落、引用块、代码块希望能像表格按行拆分那样，按「行/单元」跨页。
3. **逻辑分散**：表格拆分自成一套（`splitTableByHeight`），缺少可复用抽象。

### 1.2 设计目标

- 抽象出通用 `splitBlockByHeight(units, context, config)`，按「可拆单元 + 实测高度」贪心打包到多页。
- 各块类型（表格 / 段落 / 引用 / 代码）通过**配置 + 适配器**复用同一引擎。
- **表格现有精度与行为零回归**（首屏估算→下一帧实测、续表表头重复、容器开销分离等全部保留）。
- 引入**孤行/寡行（orphan/widow）控制**，避免单行被孤立到页首/页尾。

### 1.3 非目标（本期不做或单列）

- 不改共享渲染引擎 `src/engine/`（避免影响公众号/卡片模式）。
- **单个超长源码行**（Markdown 里写成一整行、靠自动换行折成多视觉行）的「视觉行内切分」属于 **Phase B**，复杂且较脆弱，单独评估。

## 2. 现状分析（事实依据）

### 2.1 页面几何

| 量 | 公式 | 默认值 |
|---|---|---|
| 内容区高度 `contentHeight` | `pageHeight - marginTop - marginBottom` | `1123 - 64 - 64 = 995px` |
| 可用分页高度 `effectiveHeight` | `(contentHeight - PAGE_BOTTOM_SAFETY_GAP) / scale` | `(995 - 36)/1.0 = 959px` |

- 内容区 `overflow:hidden`，超出即被裁切（[DocumentMode.tsx L336-346](file:///i:/markdown2view/src/modes/document/DocumentMode.tsx#L336-L346)）。
- 页脚绝对定位在 `bottom: marginBottom - footerHeight`，位于内容区下边界之下，**无重叠**。
- `PAGE_BOTTOM_SAFETY_GAP = 36`（[documentModel.ts L71](file:///i:/markdown2view/src/modes/document/documentModel.ts#L71)）为固定安全缓冲。

### 2.2 当前分页行为

- 主循环 [documentModel.ts L473-590](file:///i:/markdown2view/src/modes/document/documentModel.ts#L473-L590)：仅 `table` 走拆分（`splitTableByHeight`），其余块原子处理：
  ```
  if (current.length && (usedHeight + height > effectiveHeight || headingNearBottom)) pushPage()
  current.push(block); usedHeight += height
  ```
- **冗余空间主因**：原子块。次因：固定 36px 缓冲（已实测高度后偏保守）。

### 2.3 关键既有结论

- **列表已逐项成块**：`splitMarkdownBlocks` 把每个列表项拆为独立 `list` 块（[L233-249](file:///i:/markdown2view/src/modes/document/documentModel.ts#L233-L249)），所以列表**早已能逐项跨页**，本方案无需为列表新增拆分（单个超长列表项除外，归 Phase B）。
- **`avoidBreak` 是死字段**：被设置（[L258](file:///i:/markdown2view/src/modes/document/documentModel.ts#L258)）但分页函数从未引用，仅出现在测试。可在重构中赋予真实语义或移除。
- **引擎逐源码行渲染**：段落每行一个 `<section style="margin:0 0 24px"><p>`（[markdownParser.ts L710](file:///i:/markdown2view/src/engine/utils/markdownParser.ts#L710)）；引用为带边框/内距的外层 `<section>` 包裹若干 `<section><p>` 行（[L444-448](file:///i:/markdown2view/src/engine/utils/markdownParser.ts#L444-L448)）。这使「块内多行」天然是可独立实测的子元素。

## 3. 核心抽象设计

### 3.1 数据流总览

```
splitMarkdownBlocks → blocks[]
        │
        ▼
[隐藏测量容器] useBlockHeights → 每块的 { 总高, 单元高[], 非单元开销 }
        │
        ▼
paginateDocumentBlocks
        │  对「可拆块」调用
        ▼
splitBlockByHeight(units, ctx, config) → 多页片段[]
        │  每片段经 adapter.rebuild() 还原为可渲染 markdown
        ▼
DocumentPage[]
```

### 3.2 通用拆分函数签名（草案）

```ts
/** 单个可拆单元（表格的一行 / 段落的一源码行 / 代码的一行 等） */
interface SplitUnit {
  index: number        // 在块内的顺序
  height: number       // 实测高度（无实测时为估算 × heightRatio）
  groupKey?: string     // 可选：不可拆分组（如一个列表项的多行），同组必须同页
}

interface SplitContext {
  availableHeight: number   // 当前页起始可用高度
  effectiveHeight: number   // 整页可用高度（后续页）
  /** 非单元开销：包裹外边距 + 容器内边距 + 边框等（= 块总高 - Σ单元高） */
  overhead: number
  /** 续接页固定头部高度（表格表头；段落/引用为 0） */
  repeatHeadHeight: number
}

interface SplitConfig {
  /** 续接页是否重复头部（表格=true） */
  repeatHead: boolean
  /** 一页至少保留的起始单元数（防寡行：页尾不少于 N 个单元再断） */
  minHead: number
  /** 断点后下一页至少携带的单元数（防孤行） */
  minTail: number
  /** 单元间的安全缓冲 */
  safetyBuffer: number
}

interface SplitPiece {
  unitRange: [number, number]  // 本片段含的单元区间 [start, end)
  isContinuation: boolean
  height: number               // 含 overhead/head 的片段总高
}

function splitBlockByHeight(
  units: SplitUnit[],
  ctx: SplitContext,
  config: SplitConfig,
): SplitPiece[]
```

要点：
- **纯函数**，只做「按高度贪心打包 + 孤寡行修正」，不关心 Markdown 语义。
- 表格的「续表表头重复」收敛为 `repeatHead=true` + `repeatHeadHeight`。
- 段落/引用/代码：`repeatHead=false`，`overhead` 为各自包裹开销。

### 3.3 块类型适配器（Adapter）

每种可拆块提供一个适配器，负责「枚举单元」与「重建片段 markdown」：

```ts
interface BlockSplitAdapter {
  kind: DocumentBlockKind
  /** 从块 markdown + 实测单元高 → SplitUnit[] */
  toUnits(markdown: string, measured: BlockMeasurement): SplitUnit[]
  /** 把某单元区间还原为可独立渲染的 markdown 片段 */
  rebuild(markdown: string, range: [number, number], isCont: boolean): string
  config: SplitConfig
}
```

| 块类型 | 单元（unit） | overhead | rebuild 续接处理 | repeatHead |
|---|---|---|---|---|
| `table` | 数据行 `<tr>` | caption + 外边距 + 容器内距 + 边框 | 重复表头 + 「（续表）」 | **true** |
| `paragraph` | 每个源码行（`<section><p>`） | 块上下外边距 | 直接按行切，无装饰 | false |
| `quote` | 引用内每行（`<section><p>`） | `>` 外层 padding + border-left + 外边距 | 每片段重新加 `>` 前缀包裹 | false |
| `code` | 代码每行 | ``` 围栏 + pre 内距 | 每片段补 ``` 围栏（语言标记仅首片段） | false |

> 表格适配器即把现有 `splitTableByHeight` 的「行打包 + 开销分离 + 续表重建」拆成 `toUnits/rebuild/config`，**逻辑等价迁移**。

### 3.4 测量层扩展（useBlockHeights）

把现有的 `tableRowHeights / tableOverheads` 泛化为对**所有可拆块**通用：

```ts
// 现有返回
[actualHeights, tableRowHeights, tableOverheads]

// 泛化后（向后兼容，表格为其特例）
interface BlockMeasurement {
  total: number          // 块总高（含与上一块的合并外边距，逻辑同现状）
  unitHeights: number[]   // 可拆单元逐个实测高（表格=行高；段落/引用/代码=子行高）
  overhead: number        // total - Σ unitHeights
}
type Measurements = Record<string, BlockMeasurement>
```

实现要点（[useBlockHeights.ts](file:///i:/markdown2view/src/lib/useBlockHeights.ts)）：
- 对 `data-kind ∈ {table, paragraph, quote, code}` 的块，遍历其**子元素**实测 `offsetHeight` 作为 `unitHeights`：
  - 表格：沿用 `thead>tr` + `tbody>tr`。
  - 段落/引用/代码：取块内直接子 `<section>`（引擎逐行产出的行容器）。
- `overhead = total - Σ unitHeights`，与表格现有 `tableOverheads` 同义。
- 首屏无实测时，单元高用 `estimateBlockHeight` 的逐行估算 × `heightRatio`（沿用表格降级策略）。

### 3.5 孤行/寡行规则

- **寡行（widow）**：页尾若拆分后当前页单元数 `< config.minHead`，则整块下移（不在此页起拆）。
- **孤行（orphan）**：断点使下一页只剩 `< config.minTail` 个单元时，前移断点多带 1~N 个单元到下一页。
- 默认：段落 `minHead=2, minTail=2`；表格 `minHead=1, minTail=1`（保持现状，表格首页可只放 1 行数据）。

## 4. paginateDocumentBlocks 接入方式

主循环将「`if kind==='table'` 特判」替换为「`if 该 kind 有 adapter` 则走通用拆分」：

```
for block of blocks:
  if block.kind === 'pagebreak' | 'heading'(防孤标题) | oversized-image: 维持现有特殊逻辑
  else if hasAdapter(block.kind) 且 块高 > 当前剩余:
     pieces = splitBlockByHeight(adapter.toUnits(...), ctx, adapter.config)
     for each piece: 生成子块(markdown=adapter.rebuild(...)) 装入页面、必要时 pushPage()
  else:
     原子装入（短块，一页放得下）
```

- 不可拆块（`image / rule / component`）维持原子与现有「超高单独成页」逻辑。
- 封面页块（`isCoverBlock`）**一律不拆**，沿用现状。
- 子块 id 命名沿用 `${block.id}-part-${k}`，与表格一致，便于复用渲染。

## 6. 分阶段实施计划

### Phase A（低风险，先行）

1. 扩展 `useBlockHeights` 输出 `unitHeights/overhead`（泛化，表格为特例，先保证表格不回归）。
2. 实现 `splitBlockByHeight` 纯函数 + 段落/引用/代码三个 adapter。
3. `paginateDocumentBlocks` 接入通用拆分（表格暂保持调用旧 `splitTableByHeight`，**先并存**）。

5. 新增单测：长段落/引用/代码的多页拆分、孤寡行边界。

**不含**单超长折行行的视觉行切分。

### Phase B（较高风险，后续）

1. 用 `Range.getClientRects()` 测量单个 `<p>` 的视觉行盒，支持源码单行的「行内切分」。
2. 把表格 `splitTableByHeight` 正式收敛为 `table` adapter，删除并存路径与死字段 `avoidBreak`。
3. 全量回归测试表格分页（首屏估算/实测切换、续表表头、容器开销）。

## 7. 风险与回归

| 风险 | 说明 | 缓解 |
|---|---|---|
| 表格分页回归 | 表格逻辑已验证良好，是重点保护对象 | Phase A 表格**保持旧路径并存**，Phase B 才迁移并做全量回归 |
| 子元素测量不稳 | 引擎渲染结构变动会影响 `unitHeights` | adapter 的 `toUnits` 对结构做容错；无子元素时降级为整块原子 |
| 孤寡行误判 | 阈值不当导致空白或断点突兀 | 阈值入 `SplitConfig`，按块类型独立配置，便于调参 |
| 段落 rebuild 语义丢失 | 切分后样式/缩进丢失 | 段落直接按源码行 slice，不重排；引用/代码补回包裹标记 |
