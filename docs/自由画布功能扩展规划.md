# 自由画布功能扩展实施规划

## 全局架构决策

- **图片上传 UX**：采用「上传 → 复制 URL → 用户发给 AI」方案，与项目离线 AI 工作流一致，不自动修改 HTML 代码
- **风格预览缩略图**：采用纯 CSS/SVG 微型组件方案，零网络请求、体积极小，不需要维护外部截图资源
- **自定义指令持久化**：直接扩展 Zustand persist store，复用现有 `m2v-store` localStorage key
- **Tooltip 统一**：新建轻量 CSS-only Tooltip 组件 + 集中管理文案常量文件

---

## 阶段总结与后续计划 (2026-06 更新)

前期的基础功能（图片上传、Store 结构、UI 基础布局）已初步实现，但在代码审查（第一轮与第二轮）中发现了一系列严重 Bug（含 XSS 安全漏洞与数据丢失风险）以及未闭环的 UX 功能。
后续实施将重点聚焦于**修复遗留的严重问题**，补齐缺失的数据，并完成用户体验的闭环。

---

## Task 1：修复核心严重缺陷 (P0 必须立即修复)

### Task 1.1：修复 StyleThumbnail XSS 注入漏洞
- **位置**：`src/modes/html/StyleThumbnail.tsx`
- **问题**：`generateFallbackHtml()` 将 `s.name`、`s.description` 拼入 HTML 模板后通过 `dangerouslySetInnerHTML` 渲染。自定义指令允许用户输入任意文本，可能引发 XSS。
- **修复**：实现 `escapeHtml` 函数，对 `s.name` 和 `s.description` 的用户输入进行转义（处理 `<`、`>`、`&`、`"`）。

### Task 1.2：修复老用户数据 Rehydration 导致白屏/崩溃
- **位置**：`src/lib/store.ts` 和 `src/modes/html/PromptLibrary.tsx`
- **问题**：已有 `m2v-store` 的老用户，在 rehydration 时缺少 `customInstructions` 字段，会导致其为 `undefined`，进而引发 `.length` 报错，整个面板崩溃。
- **修复**：在 `onRehydrateStorage` 回调中添加兜底逻辑 `if (!state.customInstructions) { (state as any).customInstructions = [] }`。

### Task 1.3：修复超过 50 条自定义指令时静默失败的数据丢失
- **位置**：`src/lib/store.ts` 和 `src/modes/html/CustomInstructionEditor.tsx`
- **问题**：超出上限时 `addCustomInstruction` 仅 `return state`，但编辑器未收到失败反馈仍会无条件关闭，导致用户几千字的输入静默丢失。
- **修复**：让 `addCustomInstruction` 返回 `boolean`（或使用前置检查）。若超出限制，则通过 Toast 提示用户，且不关闭编辑器面板。

## Task 2：补齐核心功能与体验断层 (P1 必须交付)

### Task 2.1：实现 StylePreviewModal 的视觉渲染预览
- **位置**：`src/modes/html/StylePreviewModal.tsx`
- **问题**：全屏预览弹窗目前仅显示纯文本提示词指令，没有视觉上的"预览"，与命名语义不符。
- **修复**：在指令文本上方增加一个视觉预览区块。复用放大版的 `StyleThumbnail`（`dangerouslySetInnerHTML`）来渲染 `style.previewHtml || generateFallbackHtml(style)`。

### Task 2.2：补齐内置风格的 previewHtml 数据
- **位置**：`src/data/designPrompts.ts`
- **问题**：第一版规划要求为 primary 级别的核心内置风格编写微型 HTML 片段，但目前 `DESIGN_STYLES` 数据中完全没有 `previewHtml`。所有缩略图目前都是生成的 fallback 骨架，无法体现真实风格特征。
- **修复**：挑选具有代表性的 10-15 个核心风格，逐条编写约 15 行内的纯内联 HTML+CSS `previewHtml`，以展现其核心视觉（如字体、排版、圆角、背景）。

### Task 2.3：实现"从内置风格克隆"功能的 UI 入口
- **位置**：`src/modes/html/PromptLibrary.tsx`
- **问题**：`CustomInstructionEditor` 中已实现了 `cloneFrom` 逻辑，但在指令库 UI 中没有入口。
- **修复**：在内置风格卡片的"复制提示词"按钮旁，添加一个辅助按钮"克隆为自定义"（图标按钮即可）。点击后切换到"我的指令" Tab，并打开编辑器填入选中的 `DesignStyle`。

## Task 3：功能打磨与细节优化 (P2/P3 建议完善)

### Task 3.1：修复颜色选择器的非法 Hex 报错
- **位置**：`src/modes/html/CustomInstructionEditor.tsx`
- **问题**：用户在强调色的输入框里输入如 `#abc`、`red` 等非法色值，导致 `<input type="color">` 的受控组件报错。
- **修复**：通过正则 `/^#[0-9a-fA-F]{6}$/` 验证，若非法则传递给 `type="color"` 输入框一个 fallback 值（如 `#6366f1`），文本框保持原有显示。

### Task 3.2：完善各种 UX 状态和弹窗逻辑
- **位置**：多个组件
- **修复**：
  1. **StylePreviewModal 复制状态不重置**：增加 `useEffect`，切换风格时 `copied` 状态重置为 `false`。
  2. **UserGuidePopover 本地存储逻辑**：点击"开始使用"时，也记录到 `localStorage`，避免用户每次刷新页面都会重新看到引导浮层。
  3. **层级遮挡修复**：将 UserGuidePopover 的 `z-[100]` 降低至 `z-[35]`，确保它不会遮挡 PromptLibrary（`z-40/50`）。
  4. **PromptLibrary 性能优化**：自定义指令卡片传给 `StyleThumbnail` 的内联对象使用 `useMemo` 包裹，避免破坏子组件的 `React.memo` 优化。

### Task 3.3：跨模块文案统一与视觉强化
- **位置**：`src/modes/html/PromptLibrary.tsx`、`src/modes/html/CustomInstructionEditor.tsx`、`src/modes/html/HtmlMode.tsx`
- **修复**：
  1. 将 PromptLibrary 与 CustomInstructionEditor 中的硬编码文案替换为引用 `UI_LABELS`（如标题、Tab名称、工作流提示、保存/取消按钮等）。
  2. 为指令库的"复制提示词"按钮增加 `ring-2` 高亮特效或焦点微动画，增强引导视觉权重。
  3. 修复 `HtmlMode.tsx` 工具栏中被意外移除的 flex 对齐属性（如 `justify-end`）。
