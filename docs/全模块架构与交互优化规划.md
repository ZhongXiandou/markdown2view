# 全模块架构与交互优化规划

## 背景与目标

当前 `markdown2view` 系统中存在四个独立渲染模块（公众号、社交卡片、A4 文档、自由画布）。由于历史迭代原因，它们各自实现了相似的功能区（例如顶部导出、设置、指令工具栏），导致文案、图标、交互和提示体验的不统一。
此外，目前仅有自由画布（HTML）模式支持完善的“风格指令库”与“自定义指令”管理，其他模式仅提供简单的单行指令复制功能，用户无法在本地保存专门针对公众号或卡片的 AI 指令。

为了提供无缝、统一且专业的跨模式体验，我们制定了本次全方位的全局改造计划。

---

## 阶段一：抽象公共工具栏组件 (PreviewToolbar)

- **核心痛点**：消除各模式中（如 `ArticlePreview.tsx`, `CardMode.tsx`, `DocumentMode.tsx`, `HtmlMode.tsx`）硬编码的 `sticky top-0 z-10` 工具栏。
- **架构方案**：
  1. 在 `src/components/layout/PreviewToolbar.tsx` 中创建通用视图组件。
  2. 设计标准化的 API 接口，例如支持传入 `leftContent` 和 `rightButtons`。
  3. `rightButtons` 的结构约束为：`{ id: string, icon: string, label: string, tooltip?: string, onClick: () => void, variant?: 'default' | 'primary' | 'ghost', disabled?: boolean }`。
  4. 将现有所有操作（如“导出长图”、“复制 HTML”、“导出 PDF”、“复制富文本”）的数据流和回调通过这个组件统一调度。
  5. 统一 `Tooltip` 的方向为 `position="bottom"`，确保无视觉遮挡。

---

## 阶段二：全局化 PromptLibrary 与自定义指令

- **核心痛点**：目前“我的指令”仅存在于 `HtmlMode` 中，用户无法为文章、长文、卡片模式归档和复用高质量提示词。
- **架构方案**：
  1. **数据层改造**：
     - 修改 `store.ts` 中的 `CustomInstruction` 接口，增加核心辨识字段 `mode: RenderMode`。
     - 为了向前兼容，将现有未标注 `mode` 的历史数据平滑迁移为 `html` 模式。
  2. **内置模版抽离**：
     - 整合现有的硬编码函数（如 `buildArticleAiGuide`、`buildDocumentAiGuide`）转换为结构化的默认配置集。
     - 为 `article`、`document`、`card` 创建专属的初始化指令卡片。
  3. **视图层改造**：
     - 重构 `PromptLibrary.tsx`，将其升级为**全局路由级浮层**（或接受 `mode` 作为顶级 Prop）。
     - 当用户在对应模块点击工具栏上的“✨ 指令库”时：
       - `mode === 'html'`：加载 HTML 的 `DESIGN_STYLES` 与 HTML 类型自定义指令。
       - `mode === 'article'`：加载长图文专用的 Prompt 标签页和指令配置。
     - 复用已经修复的安全机制、XSS 防御以及容量防超限弹框。

---

## 阶段三：极简化的风格预览弹窗 (纯净的视觉反馈)

- **核心痛点**：在自由画布模式中，用户点击“预览风格”时展示了大量的系统级提示词源码（配置参数），导致用户的关注点偏离了实际视觉效果，且界面拥挤。用户只需“看效果”与“一键复制”即可。
- **优化方案**：
  1. 深度重构 `src/modes/html/StylePreviewModal.tsx`。
  2. 移除目前弹窗“右侧：配置参数”的所有代码（`<pre>{style.style}</pre>` 等底层代码块完全对用户隐匿）。
  3. 让 `previewHtml` 的真实渲染视图直接放大，置于弹窗中央或成为全视野的核心内容。
  4. 将复制指令的功能弱化代码感，放置一个巨大而醒目的行动点（CTA）大按钮：`📋 复制并去 AI 生成`。
  5. 点击该按钮后，静默复制完整的系统提示词，并弹窗提示成功，保持清爽的设计语言和顺滑的使用流。

---

## 执行预期与验收标准

1. **统一性验收**：四种模式下的工具栏必须严格对齐，具有相同高度、字体大小、图标前缀风格及 Tooltip 弹出方向。
2. **多态支持验证**：可以在任意模式下开启“指令库”，分别成功新增独立的一条自定义指令。切换模式时不会出现指令越界串台的情况。
3. **视觉聚焦验证**：在任意官方风格中点击预览，满眼只能看到设计效果，没有晦涩代码。
