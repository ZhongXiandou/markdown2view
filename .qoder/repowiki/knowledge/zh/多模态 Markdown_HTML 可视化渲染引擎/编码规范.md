- 组件定义遵循 `{Type}_{D/C}{Variant}{Number}` 命名规范（如 `Title_DA01`），并在 `engine/editor-components/index.ts` 中统一注册为 `ComponentDef` 对象，实现解析器与渲染器的解耦。
- 所有 Markdown 自定义标签（如 `<steps>`, `<p-title>`）在 `markdownParser.ts` 中通过正则匹配提取属性与内容，并调用对应组件的 `render` 方法生成带有完整内联样式的 HTML 字符串，确保导出后样式不丢失。
- 编辑器与 Store 的同步采用“本地即时更新 + 防抖回写”策略，并通过 `lastWrittenRef` 引用比对来识别并过滤掉由回写操作触发的“回声”更新，防止光标跳动或字符丢失。
- 导出功能（PNG/PDF/ZIP）统一采用“临时重置样式 -> 等待 DOM 稳定 -> 执行截图 -> 恢复原始样式”的流程，利用 `withScaleReset` 等工具函数确保截图尺寸与视觉表现的一致性。
