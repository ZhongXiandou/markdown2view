# 采用「上传即复制URL」的图片交互模式

_来源：eda6dc8 → 976215b 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
在自由画布（HtmlMode）中集成图片上传功能时，需要决定上传成功后如何处理图片引用，以适配项目「离线 AI 辅助生成 HTML」的核心工作流。

## 决策驱动
- 适配离线 AI 工作流
- 避免非技术用户的操作困惑
- 保持编辑器内容纯净

## 备选方案
- **上传后自动将 <img> 标签插入编辑器光标处** _（已否决）_ — 优点：传统编辑器的标准行为，所见即所得；缺点：在非技术用户场景下，光标位置可能不符合预期；生成的 HTML 包含本地或临时 URL，直接发给 AI 时 AI 无法访问图片，导致后续生成失败
- **上传后仅复制链接到剪贴板，由用户手动发送给 AI** — 优点：符合「用户准备素材 -> AI 生成代码」的流程；确保 AI 接收到的是可访问的 URL；不干扰编辑器当前的 HTML 结构；缺点：多一步手动粘贴操作

## 决策
选择「上传 → 复制 URL → 提示用户发送给 AI」的方案。在 `src/modes/html/HtmlMode.tsx` 中实现上传按钮，调用 `useImageUpload` Hook，成功后执行 `copyText(url)` 并弹出 Toast 提示，不自动修改编辑器内的 HTML 代码。

## 影响
用户需明确理解图片链接是作为 Prompt 的一部分提供给 AI，而非直接嵌入当前文档。若使用本地存储模式，需额外提示用户配置图床以确保 AI 可访问链接。
